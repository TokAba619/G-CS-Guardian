Video tutorial: How to create a Google Cloud Storage bucket.

Pre-flight Checklist

  • Google account with access to the target GCP project.
  • Role: roles/storage.admin (or equivalent least-privilege for bucket creation).
  • Billing enabled (required for some features).
  • Decide bucket name, region, and access model (Uniform vs Fine-grained).

Naming Tip

Format idea: csg-env-project-region e.g. csg-dev-csguardian-asia-southeast1

Step-by-Step: Create a GCS Bucket (Console)

  4. Select storage class. Standard for hot data; Nearline/Coldline/Archive for infrequent access.
  5. Access control. Prefer Uniform bucket-level access. Keep Public access blocked. Set Public access prevention to Enforced.
  6. Protection (optional). Enable Object versioning for rollbacks, add a Retention policy if needed, choose encryption (Google-managed or CMEK).
  7. Create. Click Create. The bucket appears in the list.
  8. Upload a test object. Open the bucket â†’ Upload files and add a sample file.
  9. Verify security. In Permissions, ensure no allUsers/allAuthenticatedUsers. Confirm Public access: Blocked and PAP: Enforced.
  10. (Optional) Grant IAM. Give least-privilege roles to service accounts (e.g., roles/storage.objectViewer for read-only access).
